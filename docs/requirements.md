# 要件定義: モンスト反射ルートシミュレーター

## 概要

モンスターストライク（モンスト）で「貫通タイプのキャラクターがステージ外枠の壁で反射して敵を倒す」タイプのステージ攻略を支援するWebツール。
スクリーンショットをアップロードし、キャラクターからの反射ルートを視覚的にシミュレーションできる。

## ターゲットユーザー

- 自分自身（個人利用）
- スマホで操作する

## 機能要件

### 1. スクリーンショットのアップロード

- スマホからゲームのスクリーンショットを選択・アップロードできる
- アップロードした画像をキャンバス上に背景として表示する

### 2. ステージ外枠の特定

- ステージ外枠の位置は、画面サイズに対して**固定の比率**で決まる（全ステージ共通）
- スクリーンショットの解像度から計算で求める（画像認識は不要）
- ステージ内部の障害物は対象外（将来的な拡張候補）

### 3. 手番キャラクターの位置検出

- 画面下部に4枚のキャラカードが左から1st〜4th の順で並んでいる
- そのうち**少し上に飛び出しているカード**が現在の手番キャラクター
- 手番キャラカードの検出は画像処理で自動化を目指す（カードの縦位置の差分で判定）
- ステージ上のキャラ位置は、**「1st」「2nd」等のラベル**を画像処理（テンプレートマッチング等）で検出し、ラベルからの相対位置でキャラクターの中心を特定する
- **フォールバック**: 自動検出がうまくいかない場合、ユーザーがタップでキャラ位置を手動指定できる

### 4. 反射ルートの描画

- キャラクターの中心から線を描画する
- 線はステージ外枠の壁で反射する（入射角＝反射角）
- **反射回数**: デフォルト3回。ユーザーが増減可能
- **線の長さ**: ユーザーがWeb上から変更可能

### 5. 操作UI

- **左ボタン / 右ボタン**: 線の発射角度を変更する
- **反射回数の増減**: UIから設定可能
- **線の長さの変更**: UIから設定可能
- スマホで操作しやすいサイズ・レイアウトにする

## 非機能要件

### 対応デバイス

- スマホ（iPhone）のブラウザで動作すること
- レスポンシブ対応

### デプロイ

- ローカル環境で動作すれば良い（ローカル開発サーバー）
- 将来的な外部デプロイは未定

## 技術スタック（提案）

- **フロントエンド**: React + TypeScript
- **描画**: HTML Canvas API
- **画像処理**: OpenCV.js（手番キャラクターの位置検出）
- **ビルドツール**: Vite
- **スタイリング**: Tailwind CSS

## 対象外（スコープ外）

- 敵の位置の検出・表示
- ステージ内部の障害物への反射
- ユーザー認証・データ保存
- 外部サーバーへのデプロイ
